Welcome to the ADL LRS


  • Register a user via HTTP
    • Visit our register page
    • Create a username and supply an email address with it
    • Click register!
  • Register a client via OAuth and the OAuth handshake (ALL is only scope allowed)
    • By default, only HTTP Auth is enabled. To enable OAuth, set the 'OAUTH_ENABLED' flag to true in settings.py
    • Visit our client registration page
    • Submit the name and description of the client you wish to authorize
    • Once submitted, you will receive a client ID and client secret. Use these with the appropriate OAuth headers to perform the initial handshake to gain your temporary token. (initiate) NOTE: If you are not familiar with OAuth, please consult the spec and the OAuth 1.0 RFC
    • You'll then receive your request token. Use this when visiting the authentication page. An LRS OAuth ID will be issued to you for you to POST back to the authentication page from your client
    • Once the client is authenticated, send the appropriate OAuth headers and values from your token to gain an access token.(access token)
    • You can now send this access token in with the OAuth headers when making calls to the LRS system


  • Running the LRS tests
    • Once the LRS is running, open another terminal
    • Activate your virtual env: source env/bin/activate
    • Then run the test: python manage.py test lrs
    • To run individual test modules, separate by a '.' Ex: python manage.py test lrs.StatementsTests.test_get

  • Github Examples
    • Visit our GitHub prototype page to run some of the updated .95 prototypes against the LRS
